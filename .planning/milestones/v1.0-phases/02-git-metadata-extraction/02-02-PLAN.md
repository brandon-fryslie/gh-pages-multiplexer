## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| on-disk manifest -> readManifest | Pre-existing gh-pages manifest files may be schema 1 (legacy) or schema 2 (current). Malformed schema values must not silently pass. |
| sourceRepoDir -> extractCommits | Process CWD at action start is trusted (set by actions/checkout); passing it explicitly rather than relying on ambient cwd prevents accidental drift after worktree operations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | manifest schema field | mitigate | Explicit allow-list `schema === 1 || schema === 2`; any other value throws. No silent coercion. |
| T-02-06 | Tampering | previousSha lookup | mitigate | Single `find` by `versionSlot` from in-memory parsed manifest; no string interpolation into commands. Passed as argv element to execFile in Plan 02-01's extractor. |
| T-02-07 | Denial of Service | pipeline blocks on git log | accept | git log with `-n 100` is bounded; shallow-clone failure is loud-fail (D-11), not a hang. |
| T-02-08 | Information Disclosure | dist/index.js bundles source paths | accept | Rollup config (Phase 1) disables sourcemaps. Bundle contains no dev machine paths. |
</threat_model>
