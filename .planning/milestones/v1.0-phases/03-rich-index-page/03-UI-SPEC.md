# Phase 3: Rich Index Page — UI Design Contract

**Approved:** 2026-04-06 (--auto)
**Status:** Locked

## Pillar 1: Layout

- Single-column vertical timeline. `max-width: 760px`, centered, `padding: 24px 16px`.
- Page structure (top to bottom):
  1. **Header** — repo name (h1), "{N} versions deployed" subtitle, link icon to GitHub repo URL
  2. **Versions list** — vertical stack of cards, gap `16px`, newest first
  3. **Footer** — single line, "Generated by gh-pages-multiplexer at {ISO timestamp}"
- Each version card structure:
  - Top row: version name (h2, large), timestamp (small, right-aligned, `color-fg-muted`)
  - Subtitle row: git ref (mono small) · short SHA (mono small, link to GitHub commit) · "View →" link to the deployed version
  - Optional `<details>` element labeled "{N} commits" containing a `<ul>` of commit rows
  - Each commit row: short SHA (mono) · subject line · "— author name" (muted)
- Mobile (≤600px): card padding shrinks to `16px 12px`, headings drop one size, timestamp moves to its own line.

## Pillar 3: Typography

- **Sans stack:** `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`
- **Mono stack:** `ui-monospace, SFMono-Regular, "SF Mono", Menlo, Consolas, "Liberation Mono", monospace`
- No web fonts. No font loading delay. No FOUT.
- Headings use sans stack with `font-weight: 600`. SHAs and refs use mono.

## Pillar 5: Copywriting

| String | Locked text |
|--------|------------|
| Page `<title>` | `{owner}/{repo} — Deployed Versions` |
| H1 | `{owner}/{repo}` |
| H1 link tooltip | `View on GitHub` |
| Subtitle (1+ versions) | `{N} version{s} deployed` |
| Subtitle (0 versions) | `No versions deployed yet` |
| Empty state body | `Deploy a version to see it here.` |
| Commits details summary | `{N} commit{s}` |
| View deployed link | `View →` |
| Footer | `Generated by gh-pages-multiplexer · {ISO 8601 timestamp}` |
| Commit row format | `{shortSha} {subject} — {authorName}` |
| Timestamp format | ISO 8601 with `<time datetime="..."` for accessibility; visible text is `YYYY-MM-DD HH:MM UTC` |

Tone: terse, factual, no marketing language. The page is a directory, not a landing page.

## Pillar 6: Accessibility & Interaction

- Semantic HTML: `<header>`, `<main>`, `<footer>`, `<article>` per version, `<h1>`/`<h2>` heading hierarchy, `<time datetime>` for timestamps.
- All links have visible focus rings (`outline: 2px solid var(--accent); outline-offset: 2px`).
- Color contrast meets WCAG AA: foreground tokens chosen against background tokens for ≥4.5:1 (body) / ≥3:1 (large headings).
- `<details>` element provides keyboard-accessible expand/collapse — no JS required.
- Page is fully usable with JS disabled (the renderer outputs zero `<script>` tags in v1).
- `<html lang="en">` set explicitly.
- `<meta name="viewport" content="width=device-width, initial-scale=1">` set.

## Verification gates
