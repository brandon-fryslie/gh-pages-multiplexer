**getWidgetScriptTag implementation:**

1. JSON-stringify each opts field so user-controlled values cannot break out of the script context: `const M = JSON.stringify(opts.manifestUrl); const I = JSON.stringify(opts.indexUrl); const C = JSON.stringify(opts.currentVersion);`
2. Build a single script string. **Wrap the entire body in an IIFE** so no globals leak (UI-SPEC verification gate). Structure:

```
<script>${WIDGET_MARKER}
(function(){
  const MANIFEST_URL = ${M};
  const INDEX_URL = ${I};
  const CURRENT = ${C};
  if (customElements.get('gh-pm-nav')) return;
  class GhPmNav extends HTMLElement {
    constructor(){
      super();
      const root = this.attachShadow({ mode: 'open' });
      root.innerHTML = `<style>${SHADOW_CSS}</style>${SHADOW_HTML}`;
      // ... wire up button click, fetch on first open, render rows, etc.
    }
  }
  customElements.define('gh-pm-nav', GhPmNav);
  const el = document.createElement('gh-pm-nav');
  (document.body || document.documentElement).appendChild(el);
})();
</script>
```

Error state shows "Could not load versions" — fail loud, no silent empty list.

**injectWidgetIntoHtmlFiles implementation:**

1. Compute the script tag once via `getWidgetScriptTag(opts)` — pure, no per-file recomputation.

2. Recursive walk to find every file matching `*.html` case-insensitively. Pattern lifted from `src/content-placer.ts`'s `findHtmlFiles` (the project's established pattern):

```typescript
async function findHtmlFiles(dir: string): Promise<string[]> {
  const results: string[] = [];
  const entries = await readdir(dir, { withFileTypes: true });
  for (const entry of entries) {
    const full = path.join(dir, entry.name);
    if (entry.isDirectory()) {
      results.push(...(await findHtmlFiles(full)));
    } else if (entry.isFile() && entry.name.toLowerCase().endsWith('.html')) {
      results.push(full);
    }
  }
  return results;
}
```

3. Data-driven empty case: if `htmlFiles.length === 0`, emit `core.info(\`0 HTML files in ${versionDir}, no widget injection needed\`)` and return `0`. (D-17 — no-op success, not silent skip.)

4. For each html file, run the same pipeline unconditionally:
   - `const original = await readFile(file, 'utf8');`
   - **Idempotency gate (D-12):** if `original.includes(WIDGET_MARKER)`, this file is data-marked as already-injected; the resulting `injected` value is `original` itself, and we do not write. Per [LAW:dataflow-not-control-flow] this is a data-driven selection of the new file content (already-injected → unchanged content), not a guarded skip of the *injection operation*. Implementation:
     ```typescript
     const alreadyInjected = original.includes(WIDGET_MARKER);
     const injected = alreadyInjected
       ? original
       : insertScript(original, scriptTag, file);
     if (!alreadyInjected) {
       await writeFile(file, injected, 'utf8');

     }
     ```
     Note: the write IS conditional on "did the data change?" — that's not control flow hiding work, that's avoiding a no-op fs write. The test gate (Test 12) verifies the second-pass file is byte-identical, which this satisfies.

5. **insertScript helper** — data-driven position selection (D-14):

```typescript
function insertScript(html: string, scriptTag: string, filePath: string): string {
  // [LAW:dataflow-not-control-flow] All three paths emit a string that is original + scriptTag.
  // The position of insertion varies by data (which closing tag is present), the operation does not.
  const bodyIdx = html.lastIndexOf('</body>');
  if (bodyIdx !== -1) {
    return html.slice(0, bodyIdx) + scriptTag + html.slice(bodyIdx);
  }
  const htmlIdx = html.lastIndexOf('</html>');
  if (htmlIdx !== -1) {
    return html.slice(0, htmlIdx) + scriptTag + html.slice(htmlIdx);
  }
  // Malformed HTML: no </body> AND no </html>. Append + warn (D-14).
  core.warning(`Malformed HTML in ${filePath}: no </body> or </html>; appending widget at end of file`);
  return html + scriptTag;
}
```

6. **Error propagation (D-16):** the function does NOT wrap reads/writes in try/catch. fs errors throw and bubble up. The only logging on the happy-path is the per-deploy info line and the D-14 warning. NEVER `|| true`, NEVER `2>/dev/null`, NEVER swallowed catch.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WidgetInjectionOpts → injected `<script>` tag | `currentVersion` originates from a git ref; an attacker who controls a branch/tag name could craft a value containing `</script>` or quote characters intended to break out of the script element and execute arbitrary JS in every visitor's browser. |
| versionDir filesystem → injector | HTML files inside the deployed version come from the user's source build. They may be malformed (no `</body>`, no `</html>`), already-injected (re-deploy), or contain non-HTML siblings (.css, .js, binaries). |
| branch-manager → injector (next plan) | Pipeline must trust the injector to fail loud on real errors and not silently skip work. |
