`prepareBranch(config: DeployConfig): Promise<string>`
- `workdir = path.join(os.tmpdir(), \`gh-pages-\${Date.now()}\`)`
- Configure git identity: `git config user.name "github-actions[bot]"`, `git config user.email "github-actions[bot]@users.noreply.github.com"`
- Configure remote URL with token: `git remote set-url origin https://x-access-token:${config.token}@github.com/${config.repo}.git`
- Fetch target branch: `git fetch origin ${config.targetBranch} --depth=1` (capture exit code via `ignoreReturnCode: true`)
- If fetch succeeds (exit 0): `git worktree add ${workdir} origin/${config.targetBranch}`
- If fetch fails (branch doesn't exist): create orphan branch:
  - `git worktree add --detach ${workdir}`
  - `git -C ${workdir} checkout --orphan ${config.targetBranch}`
  - `git -C ${workdir} rm -rf .`
- Return workdir

`commitAndPush(workdir: string, context: DeploymentContext, targetBranch: string, maxRetries: number = 3): Promise<void>`
- `git -C ${workdir} add -A`
- Check if there are changes to commit: `git -C ${workdir} diff --cached --quiet` (exit 0 = no changes)
  - If no changes: `core.info("No changes to deploy")` and return (no error -- idempotent)
- `git -C ${workdir} commit -m "Deploy ${context.versionSlot}"`
- Push loop (up to maxRetries attempts):
  - `git -C ${workdir} push origin HEAD:${targetBranch}` (capture exit code)
  - If exit 0: return
  - If failure and attempts remain: `core.warning(...)`, `git -C ${workdir} fetch origin ${targetBranch} --depth=1`, `git -C ${workdir} rebase origin/${targetBranch}`
- After all retries exhausted: throw Error

`cleanupWorktree(workdir: string): Promise<void>`
- `git worktree remove ${workdir} --force`

`readCnameFile(workdir: string): Promise<string | null>`
- Read `path.join(workdir, 'CNAME')` via `fs.readFile(..., 'utf8')`
- If file exists: return contents trimmed (`.trim()`)
- If file does not exist (ENOENT): return null

```typescript
import * as core from '@actions/core';
import type { DeployConfig, DeployResult, ManifestEntry } from './types.js';
import { resolveContext } from './ref-resolver.js';
import { prepareBranch, commitAndPush, cleanupWorktree, readCnameFile } from './branch-manager.js';
import { readManifest, updateManifest, writeManifest } from './manifest-manager.js';
import { placeContent } from './content-placer.js';

export function parseInputs(): DeployConfig {
  // ... (same as Plan 01 Task 2 -- already implemented, keep the export)
}

async function deploy(config: DeployConfig): Promise<DeployResult> {
  // Stage 1: Prepare branch (git worktree)
  const workdir = await prepareBranch(config);

  try {
    // Stage 2: Resolve ref context (includes CNAME detection for basePath)
    const cnameDomain = await readCnameFile(workdir);
    const context = resolveContext(config, cnameDomain !== null);
    core.info(`Version: ${context.versionSlot}, Base path: ${context.basePath}`);

    // Stage 3: Read and update manifest
    const currentManifest = await readManifest(workdir);
    const entry: ManifestEntry = {
      version: context.versionSlot,
      ref: context.originalRef,
      sha: context.sha,
      timestamp: context.timestamp,
    };
    const updatedManifest = updateManifest(currentManifest, entry);
    await writeManifest(workdir, updatedManifest);

    // Stage 4: Place content (copy + base path correction)
    await placeContent(workdir, config.sourceDir, context, config.basePathMode);

    // Stage 5: Commit and push (atomic -- content + manifest in one commit per MNFST-04)
    await commitAndPush(workdir, context, config.targetBranch);

    // Compute URL: use CNAME contents for custom domain, otherwise derive from GitHub Pages
    const baseUrl = cnameDomain !== null
      ? `https://${cnameDomain}`
      : `https://${config.repo.split('/')[0]}.github.io`;
    const url = `${baseUrl}${context.basePath}`;

    return { version: context.versionSlot, url };
  } finally {
    await cleanupWorktree(workdir).catch(e =>
      core.warning(`Worktree cleanup failed: ${e}`)
    );
  }
}

async function run(): Promise<void> {
  const config = parseInputs();
  core.info(`Deploying from ${config.sourceDir} to ${config.targetBranch}`);
  core.info(`Ref: ${config.ref}, Repo: ${config.repo}`);

  const result = await deploy(config);

  core.setOutput('version', result.version);
  core.setOutput('url', result.url);
  core.info(`Deployed ${result.version} to ${result.url}`);
}

run().catch((error: unknown) => {
  const message = error instanceof Error ? error.message : String(error);
  core.setFailed(message);
});
```

Note: The `deploy` function always runs all 5 stages. The `try/finally` ensures worktree cleanup even on error. Variability (new branch vs existing, new version vs redeploy) is in the data, not in whether stages execute. // [LAW:dataflow-not-control-flow]

IMPORTANT: The URL computation reads the actual CNAME file contents via `readCnameFile(workdir)` which returns the custom domain string (e.g., "docs.example.com") or null. This avoids the hardcoded `<custom-domain>` placeholder -- the url output will be a real URL like `https://docs.example.com/v2.1.0/`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GITHUB_TOKEN -> git remote URL | Token embedded in remote URL for authenticated push |
| gh-pages worktree -> local filesystem | Worktree content from gh-pages branch read into pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Information Disclosure | branch-manager.ts remote URL | mitigate | Token is embedded in git remote URL. Use `core.setSecret(config.token)` at start of run to mask it from logs. @actions/exec logs commands -- the token in the URL will be masked by Actions log masking. |
| T-01-09 | Denial of Service | commitAndPush retry loop | mitigate | Cap retries at 3 (hardcoded default). No exponential backoff needed -- GitHub concurrency groups (D-07) prevent most conflicts. |
| T-01-10 | Tampering | versions.json on gh-pages | accept | The gh-pages branch is writable by anyone with push access. This is inherent to the GitHub Pages model. The manifest validates schema version on read. |
| T-01-11 | Repudiation | Deploy commit authorship | accept | Commits are authored as github-actions[bot]. The triggering workflow run is linked via GITHUB_RUN_ID (logged by Actions). Sufficient for audit. |
</threat_model>
