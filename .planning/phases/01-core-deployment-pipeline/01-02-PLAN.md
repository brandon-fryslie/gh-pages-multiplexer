`sanitizeRef(ref: string): string`

`matchesPatterns(versionSlot: string, patterns: string[]): boolean`
- If patterns array is empty, return true (match all â€” per default `*` in action.yml, but empty after filtering means user explicitly wants all)
- Use `picomatch` to test versionSlot against each pattern

`resolveContext(config: DeployConfig): DeploymentContext`

- timestamp: `new Date().toISOString()`

**src/base-path.ts:**

`injectBaseHref(html: string, basePath: string, filename: string): string`
- Check for existing `<base` tag via regex `/<base\s[^>]*href="[^"]*"[^>]*>/i`
  - If found: replace its href value with basePath
  - If not found: inject `<base href="${basePath}">` after `<head>` (regex: `/(<head[^>]*>)/i`)
- After injection/replacement, fix fragment-only links: replace `href="#([^"]+)"` with `href="${filename}#$1"` globally (Pitfall 2 mitigation per research)
- If no `<head>` tag found, return HTML unchanged (don't crash on non-HTML or partial HTML)

`rewriteUrls(html: string, basePath: string): string`
- Rewrite root-relative URLs in `src` and `href` attributes: `/path` â†’ `{basePath}path`
- Match pattern: `(src|href)="(/[^"]*)"` â€” only match values starting with `/` (root-relative)
- Do NOT match protocol-relative `//` or absolute `https://` URLs
- Do NOT match already-relative paths (no leading `/`)

**src/content-placer.ts:**

`placeContent(workdir: string, sourceDir: string, context: DeploymentContext, basePathMode: 'base-tag' | 'rewrite'): Promise<void>`
- Uses Node.js `fs` module (not @actions/io) â€” this module is tested in isolation without Actions runtime
- Compute target: `path.join(workdir, context.versionSlot)`
- If target directory exists, remove it recursively (`fs.rm(target, { recursive: true, force: true })`) â€” idempotent redeploy
- Copy sourceDir to target recursively (`fs.cp(sourceDir, target, { recursive: true })`)
- Find all `.html` files in target (recursive glob)
- For each HTML file: read content, apply correction based on basePathMode:
  - `'base-tag'`: call `injectBaseHref(html, context.basePath, path.basename(htmlFile))`
  - `'rewrite'`: call `rewriteUrls(html, context.basePath)`
  - Write corrected content back
- Ensure `.nojekyll` exists at workdir root: `fs.writeFile(path.join(workdir, '.nojekyll'), '', { flag: 'a' })` (create if missing, no-op if exists)

**src/manifest-manager.ts:**

`readManifest(workdir: string): Promise<Manifest>`
- Read `path.join(workdir, 'versions.json')`
- If file doesn't exist: return `{ schema: 1, versions: [] }`
- Parse JSON. Validate schema field equals 1. If not, throw Error.

`updateManifest(manifest: Manifest, entry: ManifestEntry): Manifest`
- Pure function, no I/O
- Remove any existing entry with same `version` field (idempotent redeploy)
- Prepend new entry to versions array (newest first)
- Return new Manifest object (do not mutate input)

`writeManifest(workdir: string, manifest: Manifest): Promise<void>`
- Write `JSON.stringify(manifest, null, 2)` to `path.join(workdir, 'versions.json')`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git ref -> versionSlot | Ref name (potentially attacker-controlled) becomes filesystem directory name |
| source HTML -> injected HTML | Build output HTML gets modified with base href injection |
| versions.json -> Manifest object | JSON from gh-pages branch parsed into runtime data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | ref-resolver.ts sanitizeRef | mitigate | Strip `..` segments, null bytes, control characters. Validate result is single path segment (no `/`). Throw on empty result. Test with path traversal inputs. |
| T-01-05 | Tampering | base-path.ts injectBaseHref | accept | HTML is user-controlled build output â€” we inject a `<base>` tag, not user-supplied content. The basePath is derived from repo name + versionSlot (both sanitized). No injection vector. |
| T-01-06 | Tampering | manifest-manager.ts readManifest | mitigate | Validate `schema` field equals 1 before accepting. Reject unknown schemas rather than silently processing. |
| T-01-07 | Denial of Service | content-placer.ts glob for .html files | accept | Bounded by the user's own build output size. A user deploying their own site cannot DoS themselves. |
</threat_model>
