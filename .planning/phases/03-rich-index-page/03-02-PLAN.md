<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest.versions[*].commits[*] -> rendered HTML | Commit author names, emails, and messages are attacker-controllable (anyone who can land a commit in the deployed ref) and flow into every index page rendered by the action. |
| entry.version / entry.ref -> rendered HTML | Derived from git refs and branch/tag names; attacker-controllable via crafted refs. |
| new Date(entry.timestamp) | If an attacker manages to inject a pathological timestamp, `formatUtc` must not throw — it falls back to raw string (data-driven, still escaped). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Injection (stored XSS, version name) | renderVersionCard h2 | mitigate | escapeHtml wraps entry.version at every interpolation. Covered by Test "XSS — version name". |
| T-03-04 | Injection (stored XSS, commit message) | renderCommitRow subject | mitigate | escapeHtml wraps the split('\n')[0] subject. Body is discarded in v1 so no second injection vector. Covered by Test "XSS — commit message". |
| T-03-05 | Injection (stored XSS, author name) | renderCommitRow author span | mitigate | escapeHtml wraps commit.author_name. Covered by Test "XSS — author name" AND attribute-context spot-check. |
| T-03-06 | Injection (attribute-context) | href / title attributes | mitigate | All attribute values built from user data go through escapeHtml which escapes `"` to `&quot;`. No attributes are built from unescaped user data. |
| T-03-07 | Tampering (script injection via CSS url() or expression) | inline <style> | mitigate | Inline <style> is a static literal. No user data interpolated into CSS. |
| T-03-08 | DoS (huge manifest) | render loop | accept | Commit count is already capped at 100 per version by Phase 2 metadata-extractor. Version count is bounded by real-world deploy cadence. No additional mitigation in this phase. |
| T-03-09 | Information disclosure (author email rendered on public page) | renderCommitRow | accept | UI-SPEC explicitly shows author name only, not email (D-16 defers avatars; email is already public in git history on the source repo). Card does not render author_email. |
</threat_model>
