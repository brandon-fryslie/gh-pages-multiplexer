**Step 1 — Extend `src/branch-manager.ts`:**

Add imports at the top:
```typescript
import { writeFile } from 'node:fs/promises';
import { renderIndexHtml, type RepoMeta } from './index-renderer.js';
import type { Manifest } from './types.js';
```

Add the new export (place it after `readCnameFile` for locality with other file-I/O helpers):

```typescript
// [LAW:single-enforcer] All writes to the gh-pages worktree live in this module.
// The rendered index is produced by the pure renderer in index-renderer.ts; this
// function is the sole I/O enforcer that lands it on disk.
// [LAW:dataflow-not-control-flow] Runs unconditionally on every deploy; an empty
// manifest still produces a valid index.html (the placeholder card). The renderer
// handles empty-case rendering internally — no control-flow gate here.
export async function writeIndexHtml(
  workdir: string,
  manifest: Manifest,
  repoMeta: RepoMeta,
): Promise<void> {
  const html = renderIndexHtml(manifest, repoMeta);
  await writeFile(path.join(workdir, 'index.html'), html, 'utf8');
}
```

Do NOT modify `commitAndPush` — it already runs `git add -A` which will pick up the new file. Do NOT add any conditional around the call. The wiring happens in `src/index.ts`.

**Step 2 — Wire into `src/index.ts`:**

Import `writeIndexHtml` alongside the existing branch-manager imports. Compute `repoMeta` once:

```typescript
const [owner, repo] = config.repo.split('/');

```

Place this near the top of `deploy()` (right after `prepareBranch`, before the stages run) OR just before the `writeIndexHtml` call — either is fine, but it must be computed unconditionally and in the main path.

Then, immediately AFTER `await writeManifest(workdir, updatedManifest);` and BEFORE `await placeContent(...)`:

```typescript
// [LAW:dataflow-not-control-flow] INDX-06: index.html is regenerated on every
// deploy from the manifest. Runs unconditionally. Lands in the same commit as
// versions.json via the shared commitAndPush step (MNFST-04 / INDX-06).
await writeIndexHtml(workdir, updatedManifest, repoMeta);
```

Note: the original D-03 suggests writing index.html "after writeManifest, before the commit step." `placeContent` runs between writeManifest and commitAndPush; writing index.html before placeContent is fine (both happen in the same workdir, and `git add -A` in commitAndPush stages everything). Put the writeIndexHtml call directly after writeManifest so the two manifest-shaped writes are colocated in the source.
