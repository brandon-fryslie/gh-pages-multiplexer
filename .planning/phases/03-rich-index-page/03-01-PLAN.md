**Law annotations at the top of the file:**
- `// [LAW:one-source-of-truth]` — the manifest (versions.json) is the sole source of version data; this module is a pure projection of it to HTML.
- `// [LAW:dataflow-not-control-flow]` — same operations run every call; empty manifest and missing commits[] are encoded in the data path (array iteration over [] yields nothing; a nullish commits field becomes an empty section), not in conditional skips of operations.
- `// [LAW:single-enforcer]` — all user-controlled string interpolation MUST go through escapeHtml(). No raw `${userData}` permitted anywhere in the rendered output.

**Exports:**

```typescript
import type { Manifest, ManifestEntry, CommitInfo } from './types.js';

export interface RepoMeta {
  owner: string;
  repo: string;
}

export function escapeHtml(raw: string): string {
  return raw
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}

export function renderIndexHtml(manifest: Manifest, repoMeta: RepoMeta): string { ... }
```

**Implementation outline (skeleton only — full card body + CSS lands in 03-02):**

1. Compute derived data (pure):
   - `count = manifest.versions.length`
   - `versionWord = count === 1 ? 'version' : 'versions'`
   - `subtitleText = count === 0 ? 'No versions deployed yet' : \`${count} ${versionWord} deployed\``
   - `title = \`${repoMeta.owner}/${repoMeta.repo} — Deployed Versions\`` (em-dash U+2014, matching UI-SPEC exactly)
   - `generatedAt = new Date().toISOString()`

2. Build the HTML string via template literals. Every interpolation of user-controlled data (`repoMeta.owner`, `repoMeta.repo`, each `entry.version`, `entry.ref`, `entry.sha`, `entry.timestamp`) MUST be wrapped in `escapeHtml()`. Hard-coded literal copy (locked strings from UI-SPEC Pillar 5) is NOT wrapped — those are constants, not user input.

3. Skeleton structure:
```
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>{escaped title}</title>
  <style>/* Plan 03-02 fills this in */</style>
</head>
<body>
  <main>
    <header>
      <h1>{escaped owner/repo}</h1>
      <p class="subtitle">{subtitleText}</p>
    </header>
    <section class="versions">
      {versions.map(renderVersionCard).join('')}
      {count === 0 ? renderEmptyState() : ''}   // NOTE: acceptable here because both branches produce string content; no operation is skipped. See below.
    </section>
    <footer>Generated by gh-pages-multiplexer · {escaped generatedAt}</footer>
  </main>
</body>
</html>
```

**On the empty-state line above:** this is a data-driven selection between two strings, not a skipped operation. To stay crisp per [LAW:dataflow-not-control-flow], prefer:
```typescript
const bodyContent = count === 0
  ? renderEmptyState()
  : manifest.versions.map(renderVersionCard).join('');
```
Both sides return a string; the `section.versions` element is always emitted with a string child. No operation is skipped — the data chooses which string.

4. `renderVersionCard(entry: ManifestEntry): string` — skeleton only for this plan. Emit a stub card containing:

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Injection (stored XSS) | renderIndexHtml interpolations | mitigate | Single `escapeHtml()` helper applied at every user-data interpolation point. This plan establishes the helper and covers it with a direct correctness test (Test 9) + a version-name escape test (Test 10). Plan 03-02 extends coverage to commit messages and author names with full XSS payloads. |
| T-03-02 | Tampering | repoMeta.owner / repoMeta.repo values | mitigate | These originate from GITHUB_REPOSITORY but are still passed through escapeHtml(). Defense in depth; cost is one function call. |
